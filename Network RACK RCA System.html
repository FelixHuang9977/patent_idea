  • LLM對Network RACK log進行根因推論
  • 專家介面接收修正與標註
  • 知識圖譜持續更新

實施例：CPO Link Flap

Log: CPO port5 BER spike → UEC credit stall → GPU3 timeout

  • LLM輸出1：光纖connector鬆脫 (0.72)
  • LLM輸出2：CPO Tx laser aging (0.18)
  • 專家修正：選擇#1，標註「batch 2026-02易發生」

專家回饋介面功能

  • 呈現LLM結果＋原始log＋拓樸圖
  • 選擇正確根因或新增候選
  • 修正故障摘要與解釋
  • 標註修復步驟與拓樸context

知識更新模組

將專家回饋轉為結構化triples

  • 格式：(log_pattern, root_cause, fix_action, context)
  • 更新知識圖譜（新增節點/邊）
  • 產生fine-tune資料集提升LLM準確率

閉環推論機制

  • 未來事件優先檢索專家標註案例
  • 計算log pattern語意相似度
  • LLM結合檢索結果re-reasoning
  • 持續累積domain knowledge

專利請求項（獨立項）

  • 系統：五大模組架構
  • 方法：收集→LLM推論→專家標註→更新→閉環
  • 產品：非暫態媒體儲存程式碼

從屬項保護範圍

  • LLM prompt設計（chain-of-thought）

  • 專家介面互動（拓樸標註）
  • 特定協定log（CPO/UEC/SUE）
